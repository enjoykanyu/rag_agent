# 大模型（Large Language Model, LLM）技术概览

## 什么是大模型

大模型是指参数量达到数十亿甚至数千亿级别的神经网络模型，通常基于 Transformer 架构，通过在海量文本数据上进行预训练（Pre-training），获得强大的语言理解和生成能力。代表模型包括 GPT-4、Claude、Gemini、LLaMA、Qwen、DeepSeek 等。

## Transformer 架构核心

### 自注意力机制（Self-Attention）

自注意力机制是 Transformer 的核心创新，它允许模型在处理每个词时，同时关注输入序列中的所有位置，并根据相关性分配不同的注意力权重。

计算公式：

```
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) * V
```

其中 Q（Query）、K（Key）、V（Value）是通过输入向量分别与三个可学习的权重矩阵相乘得到的。

### 多头注意力（Multi-Head Attention）

将注意力机制并行执行多次（通常为 8-128 个头），每个头学习不同的注意力模式，最后将结果拼接并线性变换。这让模型能同时关注不同类型的信息，如语法结构、语义关联、长距离依赖等。

### 位置编码（Positional Encoding）

由于自注意力机制本身不感知序列顺序，Transformer 通过位置编码为每个词注入位置信息。原始 Transformer 使用正弦/余弦函数，现代模型多采用可学习的位置嵌入（Positional Embedding）或旋转位置编码（RoPE）。

## 预训练阶段

### 自回归语言建模（Autoregressive LM）

GPT 系列采用的方法，目标是根据前文预测下一个词。训练时最大化条件概率：

```
P(x_1, x_2, ..., x_n) = P(x_1) * P(x_2|x_1) * ... * P(x_n|x_1...x_{n-1})
```

### 掩码语言建模（Masked LM）

BERT 采用的方法，随机遮蔽输入中的部分词（通常 15%），让模型根据上下文预测被遮蔽的词。这种方式让模型能同时利用双向上下文信息。

### 训练数据与规模

现代大模型的训练数据通常包含：
- 网页文本（Common Crawl、WebText）
- 书籍和文学作品
- 学术论文和代码（GitHub、ArXiv）
- 对话数据和高质量人工标注数据

训练 token 量从数百亿到数万亿不等，参数规模从 7B 到 1.8T（如 GPT-4 据传规模）。

## 后训练阶段（Post-Training）

### 监督微调（SFT, Supervised Fine-Tuning）

使用高质量的人工标注对话数据对预训练模型进行微调，让模型学会遵循指令、进行对话。这是让模型从"续写文本"转变为"回答问题"的关键步骤。

### 人类反馈强化学习（RLHF）

1. **奖励模型训练**：收集人类对多个模型输出的偏好排序，训练一个奖励模型来预测人类偏好
2. **策略优化**：使用 PPO（Proximal Policy Optimization）等强化学习算法，让模型生成能获得高奖励分数的输出

### DPO（Direct Preference Optimization）

RLHF 的简化替代方案，直接用偏好数据优化策略模型，无需显式训练奖励模型，训练更稳定高效。

## 模型推理优化

### KV-Cache

在自回归生成过程中，缓存之前计算的 Key 和 Value 向量，避免重复计算，大幅提升生成速度。

### 量化（Quantization）

将模型权重从高精度（FP32/FP16）转换为低精度（INT8/INT4/FP8），减少显存占用和计算量：
- **PTQ（Post-Training Quantization）**：训练后量化，无需重新训练
- **QAT（Quantization-Aware Training）**：量化感知训练，精度更高但成本大
- **GGUF/GPTQ/AWQ**：针对 LLM 的专用量化方案

### 投机解码（Speculative Decoding）

使用一个小模型快速生成候选 token，再由大模型并行验证，利用大模型的并行计算能力加速生成。

## 长上下文处理

### 上下文窗口扩展

- **位置编码外推**：通过调整位置编码参数支持更长序列
- **NTK-aware 缩放**：非线性缩放位置编码，改善外推能力
- **YaRN / RoPE Scaling**：专门的长上下文扩展技术

### 上下文压缩与检索

- **RAG（Retrieval-Augmented Generation）**：将外部知识库与 LLM 结合，动态检索相关文档
- **长文本摘要**：对长文档进行分段摘要，压缩后输入模型
- **记忆机制**：如 Hierarchical Memory、StreamingLLM 等

## 多模态大模型

### 视觉-语言模型（VLM）

将图像编码器（如 ViT）与语言模型结合，实现图像理解、描述、问答等功能。代表模型：GPT-4V、Claude 3、LLaVA、Qwen-VL。

### 架构设计

- **Flamingo 风格**：在语言模型中插入交叉注意力层，连接视觉特征
- **LLaVA 风格**：使用线性投影层将视觉特征映射到语言模型的嵌入空间
- **统一编码器**：如 Gemini，使用统一的 Transformer 处理多种模态

## Agent 与工具使用

### ReAct 框架

Reasoning + Acting 的结合，模型通过"思考-行动-观察"的循环与外部环境交互：

1. **思考（Thought）**：分析当前状态，决定下一步行动
2. **行动（Action）**：调用工具（搜索、计算、代码执行等）
3. **观察（Observation）**：获取工具返回的结果
4. 循环直到完成任务

### Function Calling

模型输出结构化的函数调用请求（JSON 格式），由外部系统执行后返回结果。这是现代 LLM API 的标准能力。

### 规划与反思

- **Chain-of-Thought（CoT）**：引导模型逐步推理，"Let's think step by step"
- **Tree of Thoughts（ToT）**：维护多个推理路径，通过评估选择最优路径
- **Self-Reflection**：让模型评估自己的输出并修正错误

## 模型评估

### 标准基准测试

- **MMLU**：多学科知识问答，涵盖 57 个学科
- **GSM8K**：数学应用题，测试推理能力
- **HumanEval**：代码生成能力测试
- **HellaSwag / ARC / TruthfulQA**：常识推理和事实准确性

### 评估维度

- **知识能力**：事实准确性、领域覆盖
- **推理能力**：逻辑推理、数学计算、代码生成
- **指令遵循**：按照用户要求格式化输出、完成特定任务
- **安全性**：避免生成有害内容、保护隐私
- **多语言能力**：跨语言理解和生成

## 部署与工程实践

### 推理框架

- **vLLM**：基于 PagedAttention 的高吞吐推理引擎
- **TensorRT-LLM**：NVIDIA GPU 优化推理
- **llama.cpp**：CPU 和边缘设备上的高效推理
- **Text Generation Inference（TGI）**：Hugging Face 的生产级推理服务

### 服务架构

- **批处理（Batching）**：动态批处理请求，提高 GPU 利用率
- **连续批处理（Continuous Batching）**：vLLM 等框架支持在生成过程中动态加入新请求
- **模型并行**：张量并行（Tensor Parallelism）和流水线并行（Pipeline Parallelism）支持大模型多卡部署

## 前沿发展方向

### 模型架构创新

- **Mamba / State Space Models**：线性复杂度的序列建模，替代 Attention 机制
- **Mixture of Experts（MoE）**：稀疏激活，用更少的计算获得更大的模型容量
- **RetNet / RWKV**：兼具 Transformer 性能和 RNN 效率的新架构

### 高效训练

- **数据质量优于数量**：研究表明高质量小数据集可能优于低质量大数据集
- **课程学习**：按难度递增的顺序训练数据
- **合成数据**：使用模型自身生成高质量训练数据

### 可解释性与对齐

- **Mechanistic Interpretability**：通过分析神经元和注意力头理解模型内部机制
- **Constitutional AI**：让模型根据一套原则自我修正，减少对人类反馈的依赖
- **Red Teaming**：系统性测试模型漏洞，提升安全性
